# Pitfalls

Things that do not work, subtle bugs, and non-obvious constraints.
Read this file carefully before making changes in affected areas.

- RecognitionService cannot be annotated with @AndroidEntryPoint (Hilt). Manual DI via Application class is required.
- `GET /v1/models` is not universally supported. GROQ returns it, some local models (Ollama/LM Studio) may not. Always handle 404 by falling back to free-text model input.
- The `AudioRecord` thread must run at `THREAD_PRIORITY_URGENT_AUDIO` to avoid buffer underruns on slower devices.
- `callback.results()` must be called from the same thread that received `onStopListening()` — use `runOnUiThread` or Main dispatcher.
- WAV header requires little-endian byte ordering. Standard `DataOutputStream.writeInt()` is big-endian and will produce corrupt WAV files.
- Retrofit instances for dynamic base URLs must be cached. Creating a new Retrofit instance per call leaks OkHttp connection pools.
- `SpeechRecognizer.RESULTS_RECOGNITION` expects `ArrayList<String>` — plain `List<String>` causes a ClassCastException in Bundle.
- `settings.gradle.kts` must use `dependencyResolutionManagement` (not `dependencyResolution`) for Gradle 8.x.
- `runCatching { }` is not a coroutine scope — suspend functions cannot be called inside it. Use `withContext(Dispatchers.IO)` instead.
- `RecognitionService.Callback.readyForSpeech()` requires a `Bundle` parameter (pass `Bundle.EMPTY`) in API 30+.
- The STT `response_format=verbose_json` returns both `text` and `language` fields in one call — prefer it over separate calls for language detection.
- Kotlin 1.9.x compiler cannot read Kotlin 2.1.x stdlib metadata. When test dependencies pull newer Kotlin stdlib transitively, upgrade the project Kotlin plugin to match.
- `PromptStore` must initialize `gson` before `defaults`. Kotlin initializes properties in declaration order. A `loadDefaults()` call that references `gson` after its declaration hits an uninitialized property → NPE → swallowed by catch → fallback defaults used silently.
- `SttProviderConfig` and `LlmProviderConfig` data classes have 5 constructor params with `id` as the only default. In Kotlin 2.1.x, positional args with default params at the front require named args for the rest.
- The `ChatResponse` constructor takes `List<ChatChoice>`, not `List<ChatMessage>`. Each `ChatChoice` wraps a `ChatMessage`.
- `captureNullable<ChatRequest>()` requires an explicit type parameter in the slot approach. Prefer `slot<ChatRequest>()` for capture semantics.
- `TextInputLayout.error` persists indefinitely until explicitly set to `null`. If no `TextWatcher` clears it on text change, the red error state remains even after the user corrects the input. Always pair `textField.addTextChangedListener { layout.error = null }` when setting `layout.error` from validation logic.
- R8 full mode strips generic type information from the `Continuation<? super Response<T>>` parameter in Kotlin suspend functions within Retrofit interfaces. `HttpServiceMethod.parseAnnotations` casts to `ParameterizedType` and crashes with `ClassCastException: java.lang.Class cannot be cast to java.lang.reflect.ParameterizedType`. Use non-suspend `Call<T>` return types in code paths that run on `Dispatchers.IO` to avoid the `Continuation` type resolution entirely.
- AGP 8.2.2 bundles R8 that cannot parse Kotlin 2.1.x metadata. The R8 warning "An error occurred when parsing kotlin metadata" appears on every release build. Minimum AGP version for Kotlin 2.1 is 8.6, minimum R8 is 8.6.17. Upgrade AGP to 8.7+.
- Kotlin 2.1.x makes `javaClass.classLoader` return type `ClassLoader?` (nullable). Calling `.getResource()` without `?.` produces a compilation warning. Use `javaClass.classLoader?.getResource(…)`.
- `ResponseBody.create(MediaType?, String)` is deprecated in OkHttp 4.12.0, but the replacement extension `String.toResponseBody(MediaType?)` does not exist in this version. Suppress the warning with `@Suppress("DEPRECATION")` rather than using the unavailable extension.
- OpenRouter's `/v1/models` endpoint is unauthenticated — returns model list for any token (even garbage). The validate flow shows a green checkmark for an invalid token. Always follow model fetch with a minimal chat API call that checks for 401/403 before confirming the token works.
- `ChatRequest` had no `maxTokens` field. The token test call needs `max_tokens=1` to minimize cost. Added as optional `maxTokens: Int? = null` with `@SerializedName("max_tokens")` — existing callers are unaffected since the field defaults to null (omitted from JSON).
- R8 full mode inlines private methods and reorders instructions within `onCreate()`. `lateinit var` view fields initialized via `findViewById` in a separate `bindViews()` method may be accessed in `loadSettings()` BEFORE `bindViews()` assignments execute. Use `val by lazy { findViewById(...) }` instead of `lateinit var` + separate init method.
- `isShrinkResources = true` removes views from layouts in release builds even when the view is referenced in Java/Kotlin code via `findViewById`. Keep `isShrinkResources = false` unless resource keep rules are explicitly verified.
- `android.R.drawable.ic_media_stop` does not exist in the Android SDK. Create a custom `VectorDrawable` for a stop/square icon.
- OxygenOS (OnePlus) overrides system drawable icons — `ic_menu_preferences` renders as a wrench, not a gear. Use custom drawables for guaranteed appearance across OEMs.
- Model save validation must reject empty model text when the model list is non-empty. The check `model.isNotEmpty() && model !in models` short-circuits on empty, allowing blank model names to be saved, silently overwriting valid STT/LLM config.
- `ArrayAdapter.clear()` and `addAll()` modify the internal `mOriginalValues` list (not `mObjects`) when `mOriginalValues` is non-null — which it always is after constructor initialization. A custom `Filter.publishResults()` that calls `clear()` + `addAll()` never updates the displayed data (`mObjects`), only corrupts `mOriginalValues`. Use `BaseAdapter` + `Filterable` with a directly owned display list. Never use `clear()`/`addAll()` in a custom ArrayAdapter filter.
- `ArrayAdapter.setNotifyOnChange(false)` is never reset to `true` after a filter's `publishResults` returns, leaving the adapter in a permanently non-notifying state that causes crashes on subsequent text input.
- AutoCompleteTextView dropdown items (`DropDownListView`) cannot contain focusable child views (e.g. `ImageButton` with `focusable="true"`). The ListView refuses to deliver `onItemClick` for any row that has a focusable child — even tapping the non-focusable text area of that row fails. Keep dropdown items as plain `TextView` or use views with `focusable="false"`. For secondary actions, move controls outside the dropdown (separate dialog, long-press, etc.).
- Device rotation destroys and recreates the Activity by default. If a recording Activity uses `AudioRecord`, `onDestroy()` → `audioRecorder.cancel()` discards the PCM buffer. Always add `android:configChanges="orientation|screenSize"` to the `<activity>` element in the manifest for any Activity that handles recording.
- Release builds sign with `signingConfigs.release` which reads `app/release.keystore` (local copy of `~/.android/debug.keystore`). CI decodes `RELEASE_KEYSTORE` secret (same keystore) into `app/release.keystore`. Both use the same key — no signature mismatch.
- `installRelease` disappears as a Gradle task when the `release` build type has no `signingConfig` set. The Android Gradle Plugin does not generate the install task for unsigned release builds. Always ensure `signingConfig = signingConfigs.*` is set on the release build type, or use `-Pandroid.injected.signing.*` properties.
- `versionCode` must be a positive integer. The awk formula `printf "%d%02d%02d00"` produces `0` for `v0.0.0` because `0000000` parses as `0`. Use `01` suffix instead of `00` (`printf "%d%02d%02d01"`) to guarantee a positive result.
- AGP 9.x drops the `org.jetbrains.kotlin.android` plugin — it is no longer required and must be removed from `plugins` blocks. The `kotlinOptions {}` configuration block is also unavailable in AGP 9.x; JVM target is derived from `compileOptions { targetCompatibility }`.
- The `r0adkll/upload-google-play` GitHub Action returns a generic "Precondition check failed" error with no details when Play Console blocks a commit. Use the direct Play Developer API (via `scripts/query-play-console.py` style calls) instead to get precise error messages.
- Android VectorDrawable (`<vector>`) does not support SVG `<filter>`, `<pattern>`, `<clipPath>`, or `<feDropShadow>` elements. Gradients, drop shadows, mesh patterns, and glow effects from a high-quality SVG cannot be directly ported — render to PNG instead.
- Local `./gradlew assembleRelease` signs with the debug keystore (`app/release.keystore`, SHA1: `EE:C4:BF:...`). CI `release.yml` signs with the upload keystore (SHA1: `E2:34:27:...`). These are different keys. Play Console rejects AABs signed with the wrong key. Always use the `release.yml` workflow (triggered by `v*` tags) for Play Store uploads, not locally built AABs.
- Whisper models (especially v3) can hallucinate text like "Thank you." or "Subtitles by Amara" during silent periods. Add explicit LLM instruction guardrails in the system or user prompt to filter these out.
- The global `UncaughtExceptionHandler` in `PolishedRecognitionApp` launches `CrashDialogActivity` in a separate `:crash` process via `Intent.FLAG_ACTIVITY_NEW_TASK`. If the crash occurs on the main thread during Activity lifecycle (e.g. `onCreate`), the intent may not be delivered before the system kills the process. The `:crash` process isolation is a best-effort mitigation — the dialog may not always appear for crashes during early app startup.
- F-Droid `AutoUpdateMode: VersionTag` is NOT a valid value. Valid values: `None`, `Version`, `VersionCode`.
- F-Droid `UpdateCheckData` must have exactly 4 pipe-separated parts. Extra pipes cause `ValueError: too many values to unpack (expected 4)` in `fdroid lint`.
- Backslashes in F-Droid `UpdateCheckData` regex (`\d`, `\s`) must be preserved. Using `sed` for replacement eats the backslashes. Use Python or heredoc instead.
- `fdroid lint` passes but `fdroid rewritemeta` and `fdroid build` may fail on unrelated apps in the same MR branch. This happens when the fork has additional metadata changes (e.g. a second app). Remove all files except the target app's metadata from the branch.
- fdroid/fdroiddata upstream master does NOT contain `at.priv.graf.zazentimer.yml` (MR !39945 is not yet merged). Including it in a polished-recognition MR branch causes the maintainer to reject with "Add only one app in one MR."
